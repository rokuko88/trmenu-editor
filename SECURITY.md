## 报告安全问题

如果你发现了安全漏洞，请**不要**公开发布 Issue。

### 报告方式

1. 通过 GitHub 的 [Security Advisories](../../security/advisories/new) 私密报告
2. 或者发送邮件到维护者邮箱

### 需要提供的信息

- 漏洞的详细描述
- 复现步骤
- 影响范围
- 可能的修复方案（如果有的话）

### 处理流程

1. 我们会在 **48 小时内** 确认收到报告
2. 评估漏洞的严重程度和影响范围
3. 开发修复补丁
4. 发布安全更新
5. 公开披露漏洞详情（在修复后）

## 安全相关说明

### 数据存储

- 本项目是纯前端应用，所有数据存储在浏览器本地
- 不涉及服务器端数据传输
- 不收集任何用户数据

### 依赖管理

- 定期更新依赖包
- 使用 GitHub Dependabot 监控安全漏洞
- 及时修复已知的安全问题

### 最佳实践

使用本项目时的安全建议：

- 定期更新到最新版本
- 注意不要在公共配置文件中包含敏感信息
- 使用 HTTPS 部署生产环境
